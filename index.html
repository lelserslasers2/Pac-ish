3. Architecture of My Game

Classes: I have 3 classes, a Thing class, a Ghost class and a Coin class. The Ghost and Coin classes are subclasses of the Thing class. I had to do it that way so that the ArrayList would hold the drawable objects (see next 2 questions). The Ghost class holds its functions for chasing the pacman. The coin is made of 9 squares/pieces that are the same size as the pacman. The 'head' piece of the coin (the top-left square) is from the Coin class, but the other 8 are from the Thing class. Functions: Because the idea of this game was when I first Started writing C++ code, it was designed without classes, just functions. The functions are called as they are needed, for example the kill() function is called to end the game when the logic() function detects the pacman running into a ghost, and the moveCoin() is called to move the coin to a new place when the pacman gets the coin. Most functions are called from the draw function each frame, or from the logic() function as things happen.

4. Technical Implementation

I am most proud of the way I did the drawing of each object/character. I used an ArrayList, and added all the things I wanted to draw to it. Then I added a draw function to 'Thing' and for each item in the list, just ran the draw function. It was very simple and worked great when I had to test things. For example, I did the bad guys/ghosts/red squares last, but to draw them it was as easy as adding them to the list.

5. Most Challenging Part

The most challenging part was probably fulfilling the requirements. I wanted to make this game in processing since I started with the League of Amazing Programmers, and when I started, no one mentioned I had to have instructions (I never have those) or that I had to have 3 classes. This was a problem because I used an ArrayList of all the objects to draw and then in the draw method, I just went through the list to draw them. ArrayLists don't like mixing types, so I had to do 'extends' for it to take the different classes.
